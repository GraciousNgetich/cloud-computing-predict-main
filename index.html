Onpremise to AWS

Project Description

Part 1:

Suppose a sizable organization intends to shift its data from an on-premise system to the AWS Cloud. This transition can be accomplished by creating a source connection between the organization's file server and a cloud-based object store. The File Gateway facilitates a seamless and secure transfer of data from the on-premise file server to the cloud storage.

Services Used:
AWS IAM AWS Cloud Formation VPC AWS File Gateway AWS S3 NFS File Share AWS Storage Gateway AWS Cloudwatch Alarm

Part 2:

The second part of my responsibility involved building a resilient data pipeline using Airflow. This entailed extracting, transforming, and loading data from the source system into a SQL-based database on AWS. Throughout the migration process, I utilized various AWS services within the pipeline to ensure a seamless and reliable transfer of data.

Services Used:
AWS IAM S3 AWS Lambda AWS Cloudwatch Trigger AWS EC2 Docker Apache Airflow AWS RDS PostgreSQL AWS SNS


By leveraging these technologies and services, I successfully completed the migration of the on-premise application to the AWS cloud, establishing an efficient data pipeline in the process.
